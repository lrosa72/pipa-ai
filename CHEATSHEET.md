## ⚠️ 危险信号检查表

### 你在微操的信号

- [ ] 你说了"我觉得你应该..."
- [ ] 你要求看代码的中间版本
- [ ] 你纠正了AI的实现细节
- [ ] 你帮AI debug了一个技术问题
- [ ] 你要求修改一个函数名或变量名

**如果勾选超过2个 → 立即停止，深呼吸，回到结果导向**

### AI需要更多权限的信号

- [ ] AI不断问你"我可以...吗？"
- [ ] AI说"我需要你的帮助来..."
- [ ] 执行速度很慢（等待批准）
- [ ] AI在低风险操作上也犹豫不决

**如果勾选超过2个 → 明确给予权限，设定升级条件**

### 目标设定错误的信号

- [ ] 你无法在1秒钟内判断是否达成
- [ ] 目标是"优化"、"改进"、"看看"这类模糊词
- [ ] 没有明确的deadline
- [ ] 关注的是手段而非结果

**如果勾选超过2个 → 重写目标，使用结果导向公式**

## 🐛 故障排除

### Echo不进入角色

**症状**：Echo用"用户"称呼你，或说"作为AI助手..."

**解决**：
1. 检查配置文件路径是否正确
2. 让Echo重新读取 `core-identity-echo.md` 和 `role-echo.md`
3. 明确说："请以Echo的身份回应，不是作为AI助手"

### Elon代码质量不高

**症状**：代码有bug，或方案不够优雅

**解决**：
1. 检查是否给予了足够的上下文（项目结构、技术栈）
2. 确保Elon调用了Sub-Agent（架构师先设计，审查员把关）
3. 明确验收标准（性能指标、代码覆盖率等）

### Yuci市场效果不佳

**症状**：内容没人看，社区没增长

**解决**：
1. 给Yuci更多产品背景（为什么这个产品重要）
2. 确保内容创作者有具体的人设和风格指南
3. 让数据分析师先做受众调研

### 整体效率低下

**症状**：进展慢，AI频繁找你

**检查**：
1. [ ] 目标是否清晰？（结果导向）
2. [ ] 权限是否足够？（最高权限）
3. [ ] 你是否在微操？（不干涉过程）
4. [ ] Sub-Agent是否启用？（并行执行）

## 💡 高级技巧

### 1. 意图预加载

每周开始时，给Echo一个"战略上下文"：
```
"这周我们的战略重点是[重点]。所有子目标都应该支持这个方向。"
```

### 2. 失败预算

明确告诉AI允许试错：
```
"这个目标允许最多[2次]重大调整。不要害怕尝试，即使失败也能学习。"
```

### 3. 竞争机制

对于创造性任务，可以让多个Sub-Agent并行：
```
"让2个内容创作者各写一篇博客，然后审查员选出更好的。"
```

### 4. 长期记忆

定期让Echo总结项目上下文：
```
"总结一下我们这个月的学习，以及下个月应该保持的策略。"
```

### 5. 对抗性审查

让审查员扮演"魔鬼代言人"：
```
"审查员，请从最坏的情况审查这个方案，列出所有可能失败的地方。"
```
