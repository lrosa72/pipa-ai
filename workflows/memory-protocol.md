# 记忆协议（解决AI的失忆问题）

**系统指令：当前AI模型的上下文窗口有限，必须通过外部记忆系统持久化重要信息。每个对话开始时读取，结束时更新。**

## 记忆三层架构

```
短期记忆（当前对话） ← 中期记忆（本周任务） ← 长期记忆（项目知识）
        ↓                      ↓                      ↓
   对话历史文件        任务状态文件            架构决策/经验教训
   （易失）             （半持久）               （永久）
```

---

## 长期记忆（项目级，永久保存）

存储位置：`/memory/project/`

### 1. 架构决策记录（ADR）

**文件**：`/memory/project/architecture-decisions.md`

**目的**：记录所有重大技术决策，防止AI"忘记"为什么选这个方案。

**格式**：
```markdown
# 架构决策记录

## ADR-001: 使用Redis作为缓存层
**日期**: 2026-02-28
**决策**: 使用Redis而非Memcached
**决策者**: Elon（CTO）
**状态**: 已接受

**背景**: 认证系统需要缓存提升性能

**考虑的选项**:
- Redis: 支持持久化，数据结构丰富
- Memcached: 简单，但无持久化
- Hazelcast: 内存网格，过重

**决策理由**: 需要持久化防止缓存雪崩

**后果**: 需要维护Redis实例，增加运维复杂度

**相关代码**: `src/cache/redis-client.py`
```

### 2. 经验教训库

**文件**：`/memory/project/lessons-learned.md`

**格式**：
```markdown
# 经验教训库

## 技术教训

### LL-001: 不要过度设计缓存策略
**时间**: 2026-02-28
**场景**: 认证系统缓存层
**问题**: 初始设计考虑了10种边缘情况，代码复杂难以测试
**解决方案**: 简化到3种核心场景，用监控发现真实问题
**经验**: "先简单实现，用数据驱动优化，不要预设所有边缘情况"

## 流程教训

### LL-002: Sub-Agent并发过多导致混乱
**时间**: 2026-02-27
**场景**: 同时启动5个Sub-Agent
**问题**: Echo协调成本过高，任务冲突
**解决方案**: 限制并发数为3，引入锁机制
**经验**: "并行度不是越高越好，协调成本会指数增长"

## 常见错误

### ERR-001: 忘记更新测试
**症状**: 代码重构后CI失败
**原因**: Elon修改了函数签名但忘记更新单元测试
**预防**: 代码修改后必须搜索相关测试文件
```

## 记忆更新协议

### 对话开始时（必须执行）

1. **读取长期记忆**（1分钟）：
   ```bash
   # Echo/Elon/Yuci在开始工作前必须读取
   cat /memory/project/context-map.md
   cat /memory/project/architecture-decisions.md
   ```

2. **读取中期记忆**（30秒）：
   ```bash
   cat /memory/tasks/current-cycle.md
   cat /memory/tasks/{agent-name}-state.md
   ```

3. **读取对话摘要**（如果存在）：
   ```bash
   # 找到最新的对话摘要
   ls -t /memory/conversations/ | head -1
   ```

4. **确认理解**：
   > "我已读取项目记忆。当前我们正在进行[任务]，上次对话决定[关键决策]。今天继续？"

### 对话结束时（必须执行）

1. **更新Agent状态文件**：
   - 更新 `/memory/tasks/{agent-name}-state.md`
   - 记录当前进度、下一步、阻碍

2. **创建对话摘要**：
   - 新文件：`/memory/conversations/YYYYMMDD-HHMM-session.md`
   - 包含：关键决策、未决问题、下次预期

3. **更新经验教训**（如果有）：
   - 如果犯了错误，记录到 `lessons-learned.md`
   - 如果做了架构决策，记录到 `architecture-decisions.md`

4. **同步到当前周期记忆**：
   - 更新 `/memory/tasks/current-cycle.md` 的整体进展

## 记忆压缩策略（防止信息过载）

### 自动归档

- **对话摘要**：保留最近7天， older的移到 `/memory/conversations/archive/`
- **Agent状态**：只保留当前活跃任务，完成的任务归档
- **周期记忆**：每周结束后归档，新建下周的

### 重要性标记

在记忆文件中标记重要性：
```markdown
## [CRITICAL] 关键决策：使用Redis
这个决策影响整个缓存层，不能忘记！

## [INFO] 次要信息：开发环境Node版本
仅供参考，非关键。
```

AI优先记住标记为[CRITICAL]的信息。

## 记忆失效检测

当以下情况发生，可能是记忆失效的信号：
- AI重复问已经决策过的问题
- AI采取了与之前决策矛盾的行动
- AI忘记了明确的约束条件

**修复流程**：
1. 停止当前操作
2. 重新读取相关记忆文件
3. 确认理解："让我确认一下：我们之前决定[重述决策]，对吗？"
4. 继续执行

---

**记住：没有记忆管理的AI就像金鱼，每7秒就会忘记自己在干什么。严格执行此协议，否则系统会迅速退化为混沌。**
